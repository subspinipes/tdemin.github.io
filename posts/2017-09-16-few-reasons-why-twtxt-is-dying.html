A Few Reasons Why twtxt Is Going to Die

September 16, 2017

Timur Demin

So, the Ripper is now going for twtxt.

twtxt, at its core, is an amazingly simple microblogging platform. Its idea is to blog by simply appending lines to a publicly available text file hosted somewhere on the Internet. Simple is better, you know? Even the author of this post used to use twtxt for microblogging.

Despite its total simplicity, the population of twtxt is slowly disappearing. There are 3 latest additions vs 6 latest deletions (starting with me) on the we-are-twtxt register, and now there are just 45 guys on the list, only 10 of which occasionally tweet nowadays. Looks like a dusk for twtxt.

I think I have the points, why is this going on.

Point #1, the process of discovering existing users

As far as twtxt doc only specifies the file format for text-based microblogging, it misses lots of important things, and the first one on the list is the users discovery process.

You may say, “Ha! It’s easy to provide folks with a link on my site!”, but… exactly, you’d probably be the only one who thinks about this. Yes, just three folks from the top 10 active users on twtxt have published a link pointing to their microblog file on their sites, and those links are yet to be found by a newcomer.

There is a public registry for twtxt, but the existence of such a registry just makes the process of discovery being very hard evident.

The other side of the discovery being hard is the technical one. RSS/Atom feeds are standartized formats that are pointed to by the webpages with a special tag:

<link rel="alternate" type="application/atom+xml" href="atom.xml">

Example Atom feed link

This makes discovering the feeds by search engines and news readers very easy and streamlined, as you just enter the site URL into your favorite news reader app and a news feed is instantly found.

What can twtxt oppose RSS or Atom with? Nothing. There just isn’t a standard way to find a twtxt microblog for a crawler.

So, the only way to find guys on twtxt is a special registry. Magnificent, it’s already hard to use for a new guy.

Point #2, the quality of apps suggested for end users

Let’s imagine ourselves as we were a newcomer starting twtxt for the first time (I’m now speaking of the initial program by buckket). What command shall we issue at first? Right, it’s twtxt quickstart.

When we get to the file location chooser, we might want to store our twtxt files in a new non-existent directory (I like ~/.local/share/twtxt, but you are free to choose whatever you want). What will twtxt respond with? Right, with a crash. Nice, we’ve just started an app, and it already began crashing on quickstart.

BTW, this bug was fixed, but a version with that fix was never released, so a newbie will end up installing the program with critical bugs. Isn’t this awesome?

I haven’t checked other apps, but I’m sure there are bugs, lots of them. Can’t even imagine what would break in an average twtxt client if an official one fails on creating a directory for the twtxt file.

Point #3, the process of signing up

There actually isn’t one. Signing up is as easy as creating a publicly available text file, that’s the biggest point of twtxt. But, well, how do we exactly make a file available to anyone on the Internet?

We have a few variants to achieve this:

  • Hosting a file on something like GitHub Pages. We’ll have to sign up for GitHub first, create a repo, push our file… did I miss “learning how to work with Git”? Yeah, that was an extraordinary simple one (is this sentence even ironic?).
  • Hosting a file on a free hosting site. We’ll have to sign up on a site, learn how to work with FTP… damn it, that’s too hard again.
  • Placing a file in IPFS or KBFS. Requires some madskillz for installation and usage.

Three variants (I’m sure there are others, just not sure what are those), all of which require to install something odd, learn to work with that something odd… in other words, do those complex geeky things nobody wants to actually do.

An extra layer of complex actions required just to sign up won’t attract anyone. Just those with super powers would pass.

Point #4, and the most obvious one: the population of twtxt

OK, so we’ve come across the buggy apps and imported the we-are-twtxt registry into some app. What are we going to read now?

Did I say only those who have super powers (i.e. geeks, techies, etc) would use twtxt? This is perfectly right, there aren’t any categories of people not falling into the ones listed above. So, the timeline is flooded with the things not interesting to an average user—just things like decentralization, software, technology and stuff are discussed on twtxt. Hello? There are many more topics to discuss!

The conclusion

twtxt is not accessible for most users due to its complexity hiding in its simplicity (this looks rather strange, but it’s true). Its population is already dwindling, so the entire network is going to die after some time, even though twtxt is just 1.5 years old.

It’s a pity, but it’s true.
